# Glaucoma Intervention Analysis Pipeline

## Project Overview
Multi-agent pipeline for biostatistical evaluation of glaucoma interventions from PDF case reports. Each stage is a Claude Code subagent defined in `.claude/agents/`, orchestrated via the Claude Code `Agent` tool.

## Architecture
```
[generate-test-data] → synthetic PDFs + ground_truth.json
                            │
PDF Case Reports ───────────┘
      │
      ▼
extractor → dataset-builder → verifier → [validate vs ground truth]
                                              │
      ┌───────────────────────────────────────┘
      ▼
analyst (marimo notebook) → insights (user review) → outliner → writer ↔ critic/reviser loop → Final Report
```

## Directory Layout
```
data/pdfs/           ← Drop input PDFs here
data/extracted/      ← Per-PDF JSON extraction results
output/
  .intervention      ← Current intervention name (e.g. "SLT")
  dataset.csv        ← Compiled dataset
  dataset_verified.csv
  analysis_results.json
  insights.md        ← User-reviewable insights
  case_report_outline.md
  case_report_draft.md
  case_report_final.md
  critique.json
  validation_report.json
  validation_report.md
notebooks/
  analysis.py        ← Marimo notebook (generated by analyst agent)
.claude/agents/      ← Claude Code subagent definitions
```

## Subagents (`.claude/agents/`)

| Agent | Description |
|-------|-------------|
| `generate-test-data` | Creates 8 synthetic PDFs + `data/ground_truth.json` |
| `extractor` | pymupdf extraction → `data/extracted/*.json` |
| `dataset-builder` | Compiles JSONs → `output/dataset.csv` |
| `verifier` | Cross-checks CSV against PDFs → `output/dataset_verified.csv` |
| `validate` | Scores extraction vs ground truth → `output/validation_report.*` |
| `analyst` | Biostatistics + marimo notebook → `output/analysis_results.json` |
| `insights` | Clinical insights summary → `output/insights.md` |
| `outliner` | Case report outline → `output/case_report_outline.md` |
| `writer` | Full draft → `output/case_report_draft.md` |
| `critic` | Peer review scoring → `output/critique.json` |
| `reviser` | Revises draft based on critique → `output/case_report_draft.md` |

## Data Schema (dataset.csv)
| Column | Type | Description |
|--------|------|-------------|
| patient_id | str | Unique patient identifier |
| study_id | str | Source PDF filename (stem) |
| eye | str | OD / OS / OU |
| age | int | Patient age |
| sex | str | M / F |
| intervention | str | Intervention name |
| iop_pre_mmhg | float | IOP before intervention (mmHg) |
| iop_post_mmhg | float | IOP after intervention (mmHg) |
| followup_weeks | float | Weeks post-intervention |
| num_meds_pre | int | Glaucoma medications before |
| num_meds_post | int | Glaucoma medications after |
| diagnosis | str | Primary glaucoma diagnosis |
| notes | str | Additional clinical notes |
| source_page | int | PDF page number |
| verified | bool | Verification agent confirmed |

## Key Conventions
- Subagent definitions live in `.claude/agents/` and are run via the Claude Code `Agent` tool
- Intermediate state passes through files in `output/` — agents read predecessor outputs
- The pipeline is idempotent: re-running an agent overwrites its outputs
- Python dependencies managed by `uv`; run scripts with `uv run python <script>`
- Intervention name is stored in `output/.intervention` and read by each agent

## Running

Invoke individual agents through a Claude Code session using the `Agent` tool, or ask Claude to run the full pipeline. To run a single stage manually, start a Claude Code session and ask it to run the relevant subagent (e.g. "run the extractor agent").

```bash
# Open the interactive analysis notebook
uv run marimo edit notebooks/analysis.py
```

## Test Data Generation

The `generate-test-data` agent creates 8 synthetic PDFs:
- 5 case report–style PDFs (title, abstract, case presentation, results, discussion)
- 3 clinical note–style PDFs (EHR progress notes with exam findings)

Each has realistic IOP values, demographics, medications, and diagnoses. A ground-truth
answer key is saved to `data/ground_truth.json` so the `validate` agent can score
extraction accuracy.

## Validation

The `validate` agent compares `output/dataset_verified.csv` against `data/ground_truth.json`:
- Matches records by study_id + eye + followup_weeks
- Reports per-field accuracy (IOP pre/post, eye, diagnosis, medications)
- Outputs `output/validation_report.json` and `output/validation_report.md`

## Known Limitations
- Follow-up week conversion (month → weeks using 4.345 wks/month) can cause recall misses
  against ground truth that uses rounded integer weeks. Widen tolerance or snap common
  phrases to fix.
- The dataset contains repeated measures per eye (multiple timepoints); the analyst agent
  treats rows as independent — disclosed in the final report but not corrected statistically.
